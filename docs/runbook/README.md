# runbook/ - 운영 매뉴얼

## 역할
**장애 발생 시 대응 절차**를 문서화

## 문서 목록 (예정)
- `incident-response.md` - 알람별 대응 절차
  - 5xx 증가 → 어떻게 확인? 어떻게 복구?
  - CrashLoop → 로그 보는 법? 롤백 방법?
  - OOMKilled → 리소스 확인? 스케일 조정?
- `rollback-guide.md` - Git/Argo CD 롤백 절차
- `troubleshooting.md` - 자주 만나는 에러 해결법

## 작성 시점
- Phase 6 (모니터링) 끝나고
- Phase 7 (장애 데모) 하면서 실제 대응 흐름 문서화

## 포함할 내용
1. **징후**: 어떤 알람이 울리는가
2. **확인**: 어떤 명령어/대시보드를 보는가
3. **원인**: 일반적으로 무엇이 문제인가
4. **복구**: 어떻게 조치하는가
